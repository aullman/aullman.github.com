Now - Reliability

  • We make it our job to do browser testing.
    • WebRTC is new, things break and things change.
    • We test the latest browsers before they are released.
    • We file bugs with Chrome and Mozilla when things break (you're welcome).
    • We implement workarounds and fix things as specs change.
  • We add new features as they arrive.
  • Future proofing
    • Ironically our API has actually been around longer than the WebRTC API.
    • If we make a breaking change we will give you plenty of warning.
